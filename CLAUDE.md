# プロジェクト名

このプロジェクトは実施を行うメインツール。

## コア原則

- **シンプル第一**: 変更は最小限に。影響するコードを最小化する
- **根本原因を直す**: 一時的な回避策ではなく原因を直す
- **検証してから完了**: テスト・型チェック・実動作を確認してから完了報告する

## ワークフロー

### Planモード優先
- 3ステップ以上 or アーキテクチャに関わるタスクは Plan モードで開始
- 詰まったら無理に進めず、立ち止まって再計画する

### サブエージェント活用
- リサーチ・並列調査はサブエージェントに委譲
- メインのコンテキストをクリーンに保つ

### 完了の定義
- 動作証明できるまで完了とマークしない
- テスト実行 → ログ確認 → 実動作確認の3点セット

## 出力ルール

抽象的な分析・計画を出力したら、末尾に必ず **「次の一手」** をつける:
- 最大3項目、優先順
- 各項目に「誰が／何を／いつまでに」
- 最優先は **太字**

## 詳細ルール

- コーディングスタイル: @.claude/rules/coding-style.md
- テスト方針: @.claude/rules/testing.md

## やってはいけないこと

- `--no-verify` でフックをスキップしない
- `git push --force` を main に対して実行しない
- `.env` や認証情報をコミットしない
- 動作確認なしに「完了」と報告しない

## 開発コマンド

```bash
npm install        # 依存関係のインストール
npm run dev        # 開発サーバ起動
npm test           # テスト実行
npm run lint       # リンタ実行
npm run format     # フォーマッタ実行
```
