# Coding Style

## 命名

- 関数: 動詞始まりのキャメルケース（`fetchUser`, `parseConfig`）
- クラス: パスカルケース（`UserRepository`）
- 定数: スクリーミングスネーク（`MAX_RETRY_COUNT`）
- ファイル: ケバブケース（`user-repository.ts`）

## コメント

- 基本書かない。コードと命名で意図を表現する
- 書くのは「なぜ」その実装にしたかを残す必要があるときだけ
  - 例: 仕様の制約・パフォーマンスのための回避策・既知のバグへの対処

## 関数の長さ

- 50行を超えたら分割を検討する
- ネストは3階層以内

## エラーハンドリング

- 境界（入力受付・外部API呼び出し）でのみ try/catch する
- 内部関数では例外を握りつぶさず、上位に伝播させる
- ユーザー向けエラーメッセージと内部ログは分離する

## テスト

- ロジックを含む関数は必ずユニットテスト
- テストは Arrange / Act / Assert で構造化
- モックは境界（DB・外部API）でのみ使用。内部関数はモックしない
