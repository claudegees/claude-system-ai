# Testing Rules

## テストの種類

| 種類 | 対象 | ツール例 |
|---|---|---|
| Unit | 純粋関数・クラスメソッド | jest / vitest / pytest |
| Integration | DB・外部APIを含む処理 | supertest / testcontainers |
| E2E | ユーザーフロー | playwright / cypress |

## 実行コマンド

```bash
npm test                  # 全テスト
npm test -- --watch       # ウォッチモード
npm test path/to/file     # 特定ファイル
```

## 完了条件

タスク完了とする前に以下を必ず確認:

1. 新規追加コードに対するテストが書かれている
2. 既存テストが全て通っている
3. カバレッジが既存水準を下回っていない

## モックの方針

- **モックする**: DB / 外部HTTP API / ファイルシステム / 時刻
- **モックしない**: 同モジュール内の関数 / 純粋関数 / 設定オブジェクト

過剰なモックは「テストが通っても本番で動かない」原因になる。
ロジックは実物で、副作用のある境界だけをモックする。

## やってはいけない

- テストを skip / xit したまま放置
- アサーションのないテスト
- `expect(true).toBe(true)` 的な空テスト
- フレーキーテストを「リトライで通るからOK」とすること
