# ドメイン用語集

このプロジェクトで使う用語の定義。コード上の命名とドキュメントで一貫して使う。

## エンティティ

| 用語 | 英語 | 定義 |
|---|---|---|
| ユーザー | User | システムを利用する人。認証されたアカウント |
| アイテム | Item | 取り扱う対象物の最小単位 |
| 取引 | Transaction | アイテムの状態を変更する一連の操作 |
| カタログ | Catalog | アイテムの集合・分類体系 |
| セッション | Session | ユーザーの一連の操作の単位 |

## 状態

| 用語 | 値 | 意味 |
|---|---|---|
| 下書き | `draft` | 作成中、外部に公開されていない |
| 公開中 | `published` | 公開済み、参照可能 |
| アーカイブ | `archived` | 非表示だが履歴として保持 |
| 削除済み | `deleted` | 論理削除、復元可能 |

## 役割（Role）

| 役割 | 権限 |
|---|---|
| viewer | 閲覧のみ |
| editor | 閲覧 + 編集 |
| owner | 全操作 + 権限管理 |
| admin | システム全体管理 |

## ID 体系

- ユーザーID: `usr_xxxxxxxxxxxx`（接頭辞 + ULID）
- アイテムID: `itm_xxxxxxxxxxxx`
- 取引ID: `txn_xxxxxxxxxxxx`

## 禁則用語（使ってはいけない）

「客」「顧客」: ユーザーを「ユーザー」と呼ぶ。サービス利用者なので「顧客」は文脈不一致

「リソース」: アイテムを「リソース」と呼ばない。RESTのリソースと混同される

「マスター/スレーブ」: DB構成では「Primary / Replica」を使う
