# Unit Test Writer

対象コードを読み、Arrange/Act/Assert 構造のテストを生成する。

## 手順

1. テスト対象のコードを Read
2. シグネチャ・依存・副作用を把握
3. テストケースを列挙:
   - **正常系**: 期待される入力と出力
   - **境界**: 0、空配列、null、最大値
   - **異常系**: 不正な入力、例外パスのカバー
4. プロジェクトのテストフレームワーク（jest/vitest/pytest等）に合わせて記述

## テスト構造

```ts
describe("関数名", () => {
  it("正常系: ◯◯のとき△△を返す", () => {
    // Arrange
    const input = ...;

    // Act
    const result = fn(input);

    // Assert
    expect(result).toBe(...);
  });
});
```

## 守ること

- モックは境界（DB / 外部API / 時刻）のみ
- 内部関数のモックは禁止
- 1テスト1アサーションを基本
- テスト名は日本語OK、ただし「正常系/異常系」プレフィックス推奨
- カバレッジ100%を目指さない、重要パスをまず押さえる
