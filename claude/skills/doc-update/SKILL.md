# Documentation Updater

コードの変更に伴ってドキュメントを最新化する。

## 対象

- `README.md` のセットアップ手順
- API ドキュメント（OpenAPI / JSDoc）
- 環境変数一覧（`.env.example`）
- アーキテクチャ図（必要に応じて）
- CHANGELOG.md

## 手順

1. 直近の変更を `git log` / `git diff` で確認
2. 影響を受けるドキュメントを特定:
   - 新しい環境変数 → `.env.example` + README
   - 新しいAPI → API doc
   - 新しいセットアップ手順 → README
   - Breaking change → CHANGELOG
3. 該当箇所のみ最小限に更新
4. 古い記述（実態と合わなくなったもの）を削除

## 守ること

- **ドキュメント追加よりまず削除を検討**: 古い記述は害悪
- **コード例はコピペで動く状態**: 試さずに書かない
- **「TODO」のまま放置しない**: 書けないなら書かない
- **セクションの順序**: README は「何ができる → セットアップ → 使い方 → 開発」の順
- **絵文字は最小限**: 装飾過多は避ける

## チェックリスト

- [ ] 新規ユーザーがREADMEだけで起動できるか
- [ ] 古い API 仕様が残っていないか
- [ ] 例示コードが実際に動くか
- [ ] スクリーンショットが最新UIと一致するか
