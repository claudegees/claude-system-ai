# Debug Helper

エラー情報から根本原因を推定する。表面的な対処ではなく、**なぜ起きたか** を突き止める。

## 手順

1. エラーメッセージ・スタックトレースを取得
2. 該当行のコードを Read
3. 直前の変更を確認（`git log` / `git blame`）
4. 仮説を立てる:
   - 入力データの想定外
   - 環境差異（dev/prod）
   - 並行アクセスの競合
   - 依存関係のバージョン不整合
   - 設定ミス
5. 仮説検証のための情報収集を提案
6. 最も可能性が高い原因と修正方針を示す

## 出力形式

```
## エラー要約
1〜2行で何が起きたか

## 直接の原因
コード上のどこで失敗しているか（ファイル:行）

## 根本原因（推定）
なぜそうなったかの仮説

## 確認すべきこと
- [ ] ◯◯ をログ出力して値を確認
- [ ] △△ の設定を確認

## 修正方針
（情報が揃った後の話）
```

## 守ること

- **try/catch で握りつぶす提案をしない**: それは根本原因の隠蔽
- **「とりあえず再起動」を最初の選択肢にしない**
- **再現手順を必ず確認**: 単発か継続的かで対処が変わる
