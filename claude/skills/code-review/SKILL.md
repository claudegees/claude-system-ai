# Code Review

シニアエンジニアの視点でコードを精査する。

## 観点（severity 付き）

| 観点 | チェック内容 |
|---|---|
| **Critical** | セキュリティ脆弱性・データ損失リスク・本番停止リスク |
| **Major** | バグ・nullチェック漏れ・例外処理不備 |
| **Minor** | 可読性・命名・関数長・複雑度 |
| **Nit** | スタイル・コメントの細かい好み |

## 手順

1. `git diff` で変更を確認
2. 周辺コンテキストを Read で把握
3. 各観点で指摘を列挙
4. severity を付けて優先順位を明確化

## 出力例

```
## レビューサマリ
- Critical: 0 / Major: 2 / Minor: 3

## 指摘事項

### [Major] src/auth.ts:42
SQLインジェクション可能性。ユーザー入力を直接クエリに連結している。
推奨: parameterized query を使う

### [Minor] src/utils.ts:18
関数名 `process` が汎用すぎる。具体的な動詞に変える
```

## 守ること

- 「LGTM」だけで終わらせない
- 指摘は提案のみ、勝手にコード修正しない
- 該当しない観点はセクションごと省略OK
