# Refactor

機能を変えずにコードの可読性・保守性を改善する。

## いつ提案するか

- 同じロジックが3箇所以上に重複している
- 1関数が50行を超えている
- ネストが3階層を超えている
- 命名が抽象的すぎて意図が読み取れない
- 巨大なswitch/if-elseが分岐ごとに似た処理をしている

## 手順

1. 対象コードを Read
2. テストの有無を確認（**テストがなければリファクタ前にテスト追加を提案**）
3. リファクタ手法を選択:
   - **Extract Function**: 共通処理を関数化
   - **Rename**: 意図が伝わる名前に
   - **Replace Magic Number**: 定数化
   - **Replace Conditional with Polymorphism**: 巨大なswitch → クラス分割
   - **Inline**: 過剰な抽象化を解く
4. 1つのリファクタ = 1コミット で提案

## 守ること

- **機能変更を絶対にしない**: テストが通ることを必ず確認
- **小さい単位で**: 大きな書き換えは別タスクとして扱う
- **既存の挙動を保つ**: bugの修正は別タスク
